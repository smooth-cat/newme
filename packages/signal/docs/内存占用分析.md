## 2. 内存占用对比示例

假设我们有 **200 个节点**，每个节点：

- 被 3 个节点订阅（`subscribers`）
- 依赖 3 个节点（`dependencies`）
- **总连接数：600 条关系。**

## 方案 A：双向链表 (Link Objects)

- **结构:** 每个 `Link` 包含约 4-5 个指针属性（`upstream`, `downstream`, `prevEmitLine`, `nextEmitLine`, `prevRecLine`, `nextRecLine`）。
- **估算:**
  - 在 V8 引擎中，一个带属性的小对象约占 **32-48 字节**。
  - 600 个 Link 对象 $\times 40$ 字节 $\approx$ **24,000 字节 (24 KB)**。
  - 节点本身不需要额外的 Set 对象。

## 方案 B：两个 Set (SubSet + DepSet)

- **结构:** 每个节点持有 2 个 `Set` 对象。总共 $200 \times 2 = 400$ 个 `Set` 实例。
- **估算:**
  - 一个空的 `Set` 在内存中至少占 **60-100 字节**（不含内部存储空间）。
  - 400 个 Set 基础占用 $\approx 400 \times 80 = 32,000$ 字节。
  - **内部条目:** 每个 Set 存储 3 个引用。在 V8 中，Set 的每个 Entry 约占 **24 字节**。
  - 600 条关系 $\times 2$ (双向存储) $\times 24$ 字节 $\approx 28,800$ 字节。
  - **总计:** $32,000 + 28,800 \approx$ **60,800 字节 (60.8 KB)**。
